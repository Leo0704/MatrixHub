# MatrixHub - AI矩阵运营大师

> 这是一个用大白话写的项目说明文档

## 这玩意儿是干啥的

简单说就是个**多平台内容管理工具**。你现在要管抖音、快手、小红书三个平台对吧？每个平台都要登录、发内容、看数据，烦得很。

这个软件就是把这些重复活儿自动化，让你**一次配置，躺着发稿**。

## 能干啥

### 1. 多平台发布
把同一篇内容**一次编写，同时发布**到抖音、快手、小红书三个平台。不用一个个平台登录一个个上传了。

### 2. AI帮你写内容
接入了十几个AI模型（OpenAI、Claude、各种国产大模型等），能帮你：
- 写文案
- 生成标题
- 根据热点话题自动生成内容灵感

### 3. 热点话题监控
自动抓取各平台的热门话题，告诉你现在啥最火，方便你蹭热度。

### 4. 定时发布
提前安排好内容，定时自动发，不用守在电脑前面等。

### 5. 数据分析
把各平台的数据扒回来，看看发的东西效果咋样。

## 技术架构

### 三大部分

整个软件分三块：

| 部分 | 干啥的 |
|------|--------|
| **主进程** | 软件的后台大佬，负责所有脏活累活（数据库、AI调用、浏览器自动化） |
| **预加载** | 主进程和前端的传声筒，负责安全地传递消息 |
| **渲染进程** | 你看到的那个界面，用React写的 |

### 服务模块

```
service/
├── queue.ts              # 任务队列，管着所有要干的事
├── ai-gateway.ts         # AI网关，决定用哪个AI
├── ai-director.ts       # AI导演，指挥AI干啥活
├── platform-launcher.ts  # 浏览器控制，操控Chrome干活
├── rate-limiter.ts       # 限流器，防止请求太快被平台封
├── credential-manager.ts # 密码管家，安全存账号密码
├── monitoring.ts         # 健康检查，盯着系统状态
├── hot-topic-detector.ts # 热点探测，刷各平台的热榜
└── data-fetcher/         # 数据抓取，按平台分的爬虫
```

## 核心概念

### 任务队列

软件里所有要干的事都变成一个个**任务**，排着队等着执行。

任务有五种状态：
- `pending` - 排队等着
- `running` - 正在执行
- `completed` - 干完了
- `failed` - 失败了
- `cancelled` - 被取消了

失败了的任务会自动重试，用的策略是"越等越长"（指数退避），避免一下把人家服务器打爆。

### 浏览器池

发布内容靠的是**操控浏览器**模拟人工操作。软件维护了一个浏览器池，不用每次都重新开浏览器，省资源。

为了防止被平台检测出来是机器人，还做了各种反检测措施（隐藏自动化特征、随机操作间隔等）。

### AI网关

接了十几家AI服务商，万一某家抽风了会自动切到别家。用的设计模式叫"熔断器"，类似于电路保险丝。

每种任务可以绑特定的AI。比如写文案用Claude，生成标题用GPT-4，互不影响。

### 限流器

三个平台对请求频率都有限制。限流器帮你卡着不要超限，省得账号被封。

各平台限制大概是这样：
- 抖音：每分钟10次、每小时200次、每天1000次
- 快手：每分钟15次、每小时300次、每天2000次
- 小红书：每分钟5次、每小时100次、每天500次

## 数据库

用SQLite存数据，简单轻量。主要存这些表：

- `tasks` - 任务记录
- `accounts` - 平台账号
- `credentials` - 加密的密码
- `rate_limits` - 限流计数
- `ai_providers` - AI服务商配置
- `selector_versions` - UI元素选择器版本（平台UI变了要跟着改）
- `alerts` - 系统告警
- `metrics` - 各种指标数据

## 安全措施

- 密码不用明文存，用系统Keychain（macOS自带的安全存储）
- 进程隔离，渲染进程碰不到主进程的东西
- 敏感操作都要走IPC，不是直接调

## 常见问题

### Q: 为啥要用Electron而不是原生？
A: 主要是为了跑Playwright浏览器自动化，原生不好搞。另外React写界面快。

### Q: 为啥要自己搞浏览器自动化，不用官方API？
A: 官方API要申请、有限制、还收钱。浏览器自动化虽然黑盒，但啥功能都能用。

### Q: 数据存哪儿了？
A: 本地SQLite，在用户数据目录下，不会传到哪里去。
